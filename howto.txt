Tulkas index loader sets:
- identifier
- name
- description
- iconId and iconBackgroundId properties
- isUnique
- subCategory (int value)
- quality
- sturdiness

Iterate on tulkas items (complete index)
- find item in legacy based on id
	- found: we have both item definitions
		- check names: warn if different
		- category: legacy shall be right, take it
		- subCategory: build a map with tulkas ints and legacy strings (count each mapping)
		- unique: check and warn if different, keep tulkas value
		- sturdiness: check and warn if different, keep tulkas value
		- quality: use tulkas value
		- description: check and warn if different, use tulkas value if not empty and legacy one if tulkas is empty
		- properties: keep tulkas ones
		- keep iconURL, binding, armour value, armour type, money and bonuses from legacy.
	=> keep legacy item, update it with tulkas data
	- not found: put tulkas item in legacy map

Subcategory merge: use map to set sub category to all tulkas items
